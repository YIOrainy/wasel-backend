# Wasel – (MVP) Scope Agreement

Prepared for the UI/UX Designer

This draft is the expected scope for the MVP design of the **Wasel** mobile application design.

We want the design to be **simple, professional, modern, and very easy to use** , with a clean
user experience similar in simplicity to apps like Uber.

In the final section we share a sample screen from the Uber app as a reference for layout
simplicity and structure.

We would like to benefit from the designer experience to create a smooth and intuitive user
flow suitable for all users.

**Design Style Requirements**

- Minimal and modern design
- Clean layout
- White and black theme
- Professional look and feel
- Simple navigation
- Clear buttons and input fields
- Consistent spacing and typography
- Easy UX like Uber simplicity
- Focus on clarity and usability
- Mobile-first design
- Simple and intuitive structure

The design should feel very easy to use even for first-time users.

**Expected MVP Screens (20 screens)**

**Sender (User who sends shipment)**

1. Splash Screen
    App logo loading screen.
2. Login Screen
    Login using mobile number or email.
3. Register Screen
    Create account.
4. OTP Verification Screen
    Verification code input.
5. Home Screen
    Main screen with primary actions.


6. Create Shipment Screen
    Enter shipment details:
    pickup city, delivery city, shipment type, shipment photo, preferable date and time
    for deliver, notes bar.
7. Shipment Details Screen
    Review shipment details before submitting.
8. Offers List Screen
    List of driver offers:
    price, estimated time, rating.
9. Offer Details Screen
    Detailed view of selected offer before confirmation.
10. Shipment Tracking Screen
    Track shipment status:
    accepted, in progress, delivered, navigation map tracker.
11. Activity Screen
    Shows:
    active shipments, completed shipments, cancelled shipments, and shipment history.
12. Notifications Screen
    App notifications and updates (status of the shipment, offers, marketing).
13. User Profile Screen
    User personal information and settings.

**Captain (Driver)**

14. Driver Registration Screen
    Create driver account, Driver personal information, Vehicle type, plate number, city,
    Upload required documents.
15. Available Requests Screen
    List of available shipment requests (Active shipment request).
16. Request Details Screen
    Shipment details before submitting offer.
17. Active shipment
    Shipment details screen after accepting request.
18. Captain Activity Screen
    Shows:
    ongoing shipments, completed deliveries, previous activity, and trip history.


**Admin (Management)**

19. Admin Dashboard Screen
    Provides a general overview of platform activity, including:
    total registered users, total registered captains, number of active shipments, number
    of completed shipments, number of pending requests, recent activity summary.
20. Determine Requests (Management Screen)
    Displays a list of all shipment requests with key details:
    request ID, sender information, pickup city, delivery city, assigned captain, price,
    request status (ongoing, accepted, delivered, cancelled).

**Deliverables Required**

- Figma source file
- Clickable prototype
- Mobile app UI screens
- Consistent design system
- Typography and color style
- Clean layout structure
- Ready for developer handoff
